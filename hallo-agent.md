第一部分：智能体与语言模型基础
第一章 初识智能体
1.简介
1.1.智能体的定义：智能体被定义为任何能够通过传感器（Sensors）感知其所处环境（Environment），并自主地通过执行器（Actuators）采取行动（Action）以达成特定目标的实体。最关键的是自主性。
1.2.传统智能体的发展过程
反射智能体    没有记忆预测能力，纯靠当前状态
基于模型的反射智能体    有记忆，无目的
基于目标的智能体    主动地、有预见性地完成任务
基于效用的智能体    最大化效率的完成任务
学习型智能体    不基于人类的先验知识和设计，从环境学习
1.3.新范式
规划与推理
工具使用
动态修正
1.4.智能体的类型
  基于内部决策架构：
    见上文2
  基于时间与反应：
    1.反应式智能体：简单反应式，基于模型的智能体  速度快，开销低
    2.规划式智能体：基于目标，基于效用的智能体  有远见，速度慢
    3.混合式智能体：通过思考-行动-观察，思考阶段是规划式，行动-观察是反应式
  基于知识表示：
    1.符号主义：对于人类可读的符号进行操作和设定逻辑规则。优势是推理步骤明确，决策过程可以被完整追溯。但是缺点是在充满随机性的世界中这种设定的规则体系并不能包含所有变量，也就是知识获取瓶颈
    2.亚符号主义：神经网络和深度学习为代表，基于大量神经元，人类并不可读，也就是所谓的黑箱，它在纯粹的逻辑推理任务上表现不佳，有时会产生看似合理却事实错误的幻觉。
    3.神经符号主义：混合以上两种，完成任务时，通过大模型这种亚符号生成一系列结构化的中间步骤也就是符号实现两者的融合。

2.智能体的构成与运行原理
2.1.任务环境定义：通常使用PEAS 模型来精确描述一个任务环境，即分析其性能度量(Performance)、环境(Environment)、执行器(Actuators)和传感器(Sensors) 。
![描述](img/7d5e13ce-0f3e-4fe8-9047-1803f83c35e4.png)
环境是部分可观察的，并且分为确定性和随机性，当有其他行动者时就是多智能体环境，几乎所有任务都发生在序贯且动态的环境中。“序贯”意味着当前动作会影响未来；而“动态”则意味着环境自身可能在智能体决策时发生变化。
2.2.智能体的运行机制
并非一次性完成所有任务，而是持续的循环与环境进行交互也就是智能体循环，其中包括几个阶段
  1.感知
  2.思考：包括规划和工具选择
  3.行动
  行动后再感知形成一个持续的“感知-思考-行动-观察”的闭环
2.3.智能体的感知与行动
为了有效循环需要规定明确的交互协议，通常包括Thought，Action，Observation三个字段

3.智能体应用的协作模式
分为两种类型：
3.1.作为高效工具，深度融入工作流，例如Cursor，Claude Code等
3.2.作为自主的协作者，与其他智能体协作完成复杂目标
框架分为几个方向：
  1.单智能体自主循环
  2.多智能体协作：比如 角色扮演式对话 的CAMEL，又比如 组织化工作流 的CrewAI和MetaGPT，也可以用AutoGen 和 AgentScope等自定义智能体间的复杂交互网络
  3.高级控制流架构：比如 LangGraph 
3.3.工作流和智能体的区别
Workflow 是让 AI 按部就班地执行指令，而 Agent 则是赋予 AI 自由度去自主达成目标。
工作流的核心是对一系列任务或步骤进行预先定义的、结构化的编排
智能体是一个具备自主性的、以目标为导向的系统，包括规划与工具调用，推理与决策，生成结果。基于实时信息进行动态推理和决策的能力，正是 Agent 的核心价值所在。
